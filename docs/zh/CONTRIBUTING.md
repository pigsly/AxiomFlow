# Contributing

閱讀順序：`14 / 14`


這個專案重視的是治理品質，不只是文件數量。

如果你要貢獻內容，目標應該是提升治理模型的清晰度、控制力與可採用性。

## 你可以貢獻什麼

有價值的貢獻包括：

- 把核心概念講得更清楚
- 補上更好的範例
- 改善導入與上手文件
- 強化治理工作流程
- 提供可重用的文件模板
- 改善治理升級提案的表述方式

## 在修改治理內容前

如果你要改動會影響治理模型本身的內容，先檢查：

- 這個改動是否改變了 `REQ`、`SPEC_STEP`、`ADR`、`CONTRACT`、`REFLECT` 或 `SUGGEST` 的意義
- 這個改動是否影響 trust order
- 這個改動是否改變了停機條件或批准邊界
- 這個改動應該被表達成範例、導讀，還是治理提案

## 貢獻原則

- 優先讓規則更清楚，而不是更多
- 優先把邊界講明，而不是依賴暗示
- 優先提供能教會人關係的範例，而不只是格式
- 不要隨意削弱停機條件
- 不要把提案層和正式治理層混在一起

## 建議流程

1. 先提出 issue，或先寫下目前文件沒解好的問題。
2. 判斷這個改動是解釋性、結構性，還是治理層級的改動。
3. 更新對應文件。
4. 如果改動影響治理意義，請在 pull request 中明確說明理由。

## 模板與範例

如果你新增模板：

- 保持精簡
- 讓文件角色一眼可辨
- 不要過度設計那些未必能泛化的欄位

如果你新增範例：

- 情境要具體
- 要看得出這個範例為什麼重要
- 內容要和 `Governance.md` 的規則保持一致

# Contributing

閱讀順序：`14 / 14`


This project is about governance quality, not just document quantity.

If you contribute, the goal is to improve clarity, control, and adoption of the governance model.

## What To Contribute

Useful contributions include:

- clearer explanations of core concepts
- better examples
- improved onboarding documentation
- stronger governance workflows
- reusable document templates
- better proposal wording for rule upgrades

## Before You Change Governance Content

Before changing material that affects the governance model itself, check:

- whether the change alters the meaning of `REQ`, `SPEC`, `ADR`, `CONTRACT`, `REFLECT`, or `SUGGEST`
- whether the change affects trust order
- whether the change changes stop conditions or approval boundaries
- whether the change should be represented as an example, a guide, or a governance proposal

## Contribution Principles

- prefer clearer rules over more rules
- prefer explicit boundaries over implied intent
- prefer examples that teach relationship, not just format
- do not weaken stop conditions casually
- do not blur the difference between proposal layers and active governance

## Suggested Process

1. Open an issue or write down the problem the current documentation does not solve well.
2. Identify whether the change is explanatory, structural, or governance-level.
3. Update the relevant document.
4. If the change affects governance meaning, explain the reasoning clearly in the pull request.

## Templates And Examples

If you add a new template:

- keep it minimal
- make the role of the document obvious
- avoid over-designing fields that may not generalize

If you add a new example:

- make the scenario concrete
- show why the example matters
- keep it aligned with the rules in `Governance.md`
